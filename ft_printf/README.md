# ft_printf
## Description
---
42 Tokyo の共通コア（Common Core）課題の一つで、C言語の標準ライブラリ関数である `printf` を再実装するプロジェクトです。
可変長引数の扱いを理解し、各データ型を正確に処理して標準出力に書き出すプログラムを構築した。また本家同様に「最終的に画面に出力した文字数」を戻すように設計した。

### 📌 対応しているフォーマット指定子

本プロジェクトでは、以下の9種類のフォーマット指定子をサポートしています。

| 指定子 | 出力する内容 |
| :---: | --- |
| `%c` | 単一の文字 |
| `%s` | 文字列（`NULL` ポインタの時は `(null)` を出力） |
| `%p` | ポインタのアドレスを16進数（`0x` から始まる形式）で出力 |
| `%d` | 10進数の符号付き整数 |
| `%i` | 10進数の符号付き整数（`%d` と同様） |
| `%u` | 10進数の符号なし整数 |
| `%x` | 16進数の数値を小文字（`0123456789abcdef`）で出力 |
| `%X` | 16進数の数値を大文字（`0123456789ABCDEF`）で出力 |
| `%%` | パーセント記号（`%`）自体を出力 |

## Instructions
### 1.コンパイルとライブラリの作成
---
リポジトリのルートディレクトリで`make`を実行することで、ソースファイルがコンパイルされ、静的ライブラリ`lobfrprintf.a`が生成されます。

```bash
make
```
### Makefileの基本ルール
- `make all` ; ライブラリをコンパイル・生成します。
- `make clean` : コンパイル時に生成されたオブジェクトファイル（`.o`）を削除します。
- `make fclean` : オブジェクトファイルに加え、生成された`libftprintf.a`も完全に削除します。
- `make re` : `fclean`を実行した後に`all`を行い、完全にサイビルドします。


### 2.実行方法
生成された`libft.a`とヘッダーファイル`ft_printf.h`を自身のソースコードと同じディレクトリに配置し、コード内で以下のようにインクルードして使用します。

```C
#include "ft_ptintf.h"
int main()
{
    ft_printf("Hello %s!\n", "42 Tokyo");
    ft_printf("Hex: %x, Pointer: %p\n", 255, &main);
    return (0);
}
```
コンパイルする際は、以下のようにライブラリファイルをリンクしてください。

```Bash
cc main.c libftprintf.a
```

##  アルゴリズムとデータ構造
### 1.再帰アルゴリズムによる数値変換
数値（`%d`,`%i`,`%u`）および16進数(`%x`,`%X`)の出力において**再帰関数**を採用しています。

- **採用の理由** : 文字列を格納するためのバッファを動的に確保する手法では、桁の計算、メモリ確保の失敗への対策、出力後の解放漏れによるメモリリークのリスクが伴うから

- **メリット** : 再帰を使用することで、数値の上位桁から順に直接`write`関数（`ft_putchar）で標準出力へ流し込むことが可能になり、動的確保がひつお用ない安全な設計を実現しました。

### 2.`long`型および`unsigned long long`型によるオーバーフロ対策
- `%d` や `%i` で扱われる `int` 型の最小値（`INT_MIN: -2147483648`）は、符号を反転させようとすると int の最大値を超えるためオーバーフローが発生します。これに対処するため、内部的に `long` 型の変数にキャストして処理するデータ構造をとっています。

- ポインタアドレス（`%p`）の処理においては、環境に依存しない広大なメモリ空間のアドレス値を安全に保持するため、`unsigned long long` 型を採用して16進数変換関数へ引き渡しています。
